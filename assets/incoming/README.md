# Incoming Assets

Place owner-provided raw assets here first. Later phases will optimize and move production-ready files into the app/public asset structure.

## Naming conventions

- Logo source: `hoosh-yar-logo-source.svg` preferred. If unavailable, use `hoosh-yar-logo-source.png` or `hoosh-yar-logo-source.jpeg`.
- Hero video: `hero-video.mp4`
- Case study assets: `case-studies/<case-study-slug>/cover.*` plus supporting screenshots/video.

## Hero video expected spec

- 16:9 landscape
- 1920×1080 minimum, 3840×2160 ideal
- About 10 seconds
- Seamless loop
- `.mp4` H.264 source
- No audio required
- Aim under 8–10MB before further web optimization
